KFC套餐问题

​ 建造者模式可以用于描述KFC如何创建套餐：套餐是一个复杂对象，它一般包含主食（如汉堡、鸡肉卷等）和饮料（如果汁、可乐等）等组成部分，不同的套餐有不同的组成部分，而KFC的服务员可以根据顾客的要求，一步一步装配这些组成部分，构造一份完整的套餐，然后返回给顾客。

Builder

抽象建造者，组合了KFC套餐

ConcreteBuilder

具体建造者，继承 Builder，实现了不同的构建逻辑

Director

包工头，构建一个使用 Builder 接口的对象。它主要是用于创建一个复杂的对象，它主要有三个作用：

    1. 隔离了客户与对象的生产过程
    2. 负责控制产品对象的生产过程
    3. 传什么给包工头，就生产什么种类的KFC
